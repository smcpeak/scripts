Scott's Scripts

This is the contents of my $HOME/scripts directory, which contains a grab-bag of shell scripts I find useful.

Each script uses one of four scripting languages:

  • sh: Bourne shell, the original Unix shell. Very portable.
  • bash: The GNU Bourne-again shell, an extended version of sh. Ubiquitous on Linux, less so on other Unix variants. I've only identified a few scripts that require the bash extensions, there are probably others.
  • csh: Bill Joy's C Shell. I do not write scripts with it anymore. Very portable.
  • perl: Larry Wall's multipurpose scripting language. Fairly portable, but with annoying inconsistencies between versions that make it less portable than it might initially appear.

At this point my strategy for choosing a language is to use sh when possible, and perl when I have to. Typically, "possible" means the script is small and doesn't require much string processing. I try to avoid using the bash extensions, but since I do not have a true Bourne shell installed, they inevitably creep in and I fix them when someone complains.
